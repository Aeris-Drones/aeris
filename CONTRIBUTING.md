# Contributing to Aeris

Thanks for your interest in contributing! To keep things moving smoothly:

1. Discuss significant changes by opening an issue first.
2. Fork the repository and create feature branches from `main`.
3. Ensure code is formatted and linted before opening a pull request.
4. Include tests or simulation scenarios when adding new functionality.
5. Follow the code review checklist in `docs/ops` for safety-critical changes.

We welcome suggestions to improve these guidelines as the project matures.
